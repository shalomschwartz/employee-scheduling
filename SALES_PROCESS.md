# תהליך מכירה — ShiftSync

מדריך פנימי לשימושי בלבד. עקוב אחר השלבים בסדר.

---

## שלב 1 — שיחת היכרות

**מה לשאול:**
- כמה עובדים יש בעסק?
- איך הם מנהלים סידור עבודה היום? (וואטסאפ, אקסל, בעל פה?)
- כמה זמן זה לוקח להם כל שבוע?
- האם העובדים עם טלפון נייד?

**מה להציג:**
- הדגמה חיה של האפליקציה (כנס עם חשבון הדמו)
- הראה: עובד שולח זמינות → מנהל לוחץ "צור סידור" → PDF יוצא

## שלב 3 — הקמת סביבה ללקוח

### 3א — צור מסד נתונים

1. כנס ל־ [railway.app](https://railway.app)
2. לחץ **New Project → PostgreSQL**
3. לאחר יצירה לחץ על הפרויקט → **Variables**
4. העתק את `DATABASE_URL`

### 3ב — צור פרויקט Vercel

1. כנס ל־ [vercel.com](https://vercel.com)
2. לחץ **Add New → Project**
3. בחר את ה־repo של ShiftSync מ־GitHub
4. תחת **Environment Variables** הוסף:

```
DATABASE_URL        = [הערך מ-Railway]
NEXTAUTH_SECRET     = [מחרוזת אקראית ארוכה — לדוגמה: הרץ: openssl rand -base64 32]
NEXTAUTH_URL        = https://[שם-הפרויקט].vercel.app
```

5. לחץ **Deploy**
6. המתן לסיום — Vercel מריץ `prisma db push` אוטומטית

### 3ג — בדוק שהכל עובד

- כנס לכתובת שנוצרה
- הירשם כמנהל עם אימייל הלקוח
- צור ארגון עם שם העסק
- הוסף עובד אחד לדוגמה ובדוק שהכניסה עובדת

## שלב 4 — מסירה ללקוח

**שלח ללקוח:**

```
שלום [שם],

האפליקציה שלך מוכנה!

קישור: https://[שם-הפרויקט].vercel.app

כניסת מנהל:
אימייל: [האימייל שלו]
סיסמה: [הסיסמה שהגדרת]

לכניסה: לחץ "כניסת מנהל" בתחתית דף הכניסה.

מצורף מדריך שימוש.
```

- צרף את קובץ `CLIENT_ONBOARDING.md` (או שלח כ-PDF)
